01. 함수 : 선언적 함수

선언적 함수는 자바스크립트에서 함수를 선언할 때 이름을 붙여주는 방식입니다.
이 방식을 사용하면, 함수를 따로 변수에 할당하지 않고도 선언한 이름으로 바로 호출할 수 있어요. 📞

선언적 함수의 특징
* 명시적 이름 부여: 선언적 함수는 함수 선언 시 명시적으로 이름을 부여합니다. 이 이름을 사용하여 함수를 호출할 수 있어요.
* 호이스팅(Hoisting) 혜택: 선언적 함수는 자바스크립트 엔진에 의해 코드의 최상단으로 끌어올려지는 호이스팅의 혜택을 받습니다.
이는 함수를 선언하기 전에도 호출할 수 있게 해줍니다. 

{
    function func() {
        console.log("01. 함수 : 선언적 함수");
    };
} func();

function func() { ... }: 이 부분은 func라는 이름의 함수를 정의하는 부분입니다.
함수 내용은 중괄호 {} 안에 위치하며, 여기서는 "01. 함수 : 선언적 함수"라는 문자열을 콘솔에 출력하고 있습니다.

func(): 이 부분은 정의된 func 함수를 호출하는 부분입니다. 이렇게 호출하면 함수 내부의 코드가 실행되고,
"01. 함수가 실행되었습니다"가 콘솔에 출력됩니다.

02. 함수 : 익명 함수

익명 함수는 이름이 없는 함수를 가리킵니다.
일반적으로 함수를 정의할 때 함수의 이름을 지정하는데, 익명 함수는 이름을 가지지 않기 때문에 익명 함수라고 부릅니다.
익명 함수는 코드를 보다 간결하게 만들고, 필요한 곳에 간편하게 사용할 수 있는 장점이 있습니다 

{
    const func = function () {
        console.log("02. 함수 : 익명 함수");
    };
    func();
}

const func = function () { ... };: 이 부분은 익명 함수를 변수 func에 할당하는 부분입니다.
함수 내용은 중괄호 {} 안에 위치하며, 여기서는 "02. 함수 : 익명 함수"라는 문자열을 콘솔에 출력하고 있습니다.

func();: 이 부분은 변수 func에 할당된 함수를 호출하는 부분입니다.
변수에 할당된 함수를 호출하면 함수 내부의 코드가 실행되고, "02. 함수 : 익명 함수"가 콘솔에 출력됩니다.

03. 함수 : 매개변수 함수

매개변수(parameter)는 컴퓨터 프로그래밍 및 수학에서 사용되는 중요한 개념입니다.
매개변수는 함수나 프로시저에 입력으로 전달되는 값을 나타냅니다.
이러한 값은 함수나 프로시저가 실행되는 동안에만 유효합니다.

프로그래밍에서 매개변수는 함수에 전달되는 값들을 가리키며, 함수를 호출할 때 해당 값들을 함수 내부로 전달합니다.
매개변수는 함수의 정의에 포함되어야 하며, 함수를 호출할 때 실제로 전달되는 값과 매핑됩니다.

{
    // 선언적 함수
    function func(str) {
        console.log(str);
    };
    func("03. 함수 : 매개변수 함수");

    // 익명 함수
    const func1 = function (str) {
        console.log(str);
    };
    func1("03. 함수 : 매개변수 함수");
}

04. 함수 : 리턴값 함수

함수가 리턴값을 가질 때, 해당 함수는 호출자에게 결과를 반환합니다.
이 결과는 함수가 계산하거나 처리한 값입니다. 리턴값을 가진 함수는 주어진 입력에 대한 결과를 계산하고, 이 결과를 호출자에게 반환하게 됩니다.

프로그래밍에서 함수의 리턴값은 보통 함수의 실행 결과를 나타냅니다.
이것은 다른 함수나 변수에 할당하거나 조건문에서 사용되거나 출력되는 등 다양한 방식으로 활용될 수 있습니다. 

{
    // 선언적 함수
    function func() {
        return "04. 함수 : 리턴값 함수";
    };
    console.log(func());

    //익명함수
    const func1 = function () {
        return "04. 함수 : 리턴값 함수";
    };
    console.log(func1());
}

05. 함수 : 매개변수 + 리턴값 함수

매개변수와 리턴값을 함께 사용하는 함수는 주로 입력을 받아 처리한 후 그 결과를 반환하는데 사용됩니다.
이렇게 함으로써 함수는 외부의 값을 받아들이고, 그 값을 기반으로 계산 또는 처리를 수행한 후 그 결과를 반환합니다. 

{

    //익명함수
    const func1 = function (str) {
        return str;
    };
    console.log(func1("05. 함수 : 매개변수 + 리턴값 함수"));

06. 화살표 함수 : 선언적 함수

선언적 함수를 화살표 함수로 바꾸는 것은 JavaScript에서의 일반적인 변형 중 하나입니다.
화살표 함수는 함수를 더 간결하게 표현할 수 있고, 함수의 스코프를 쉽게 이해할 수 있도록 도와줍니다. 

{
    //선언적 함수

    function func() {
        console.log("06. 화살표 함수 : 선언적 함수");
    };
    func();

    func1 = () => {
        console.log("06. 화살표 함수 : 선언적 함수");
    };
    func1();
}

09. 화살표 함수 : 리턴값 함수

리턴값을 갖는 함수를 화살표 함수로 변환하는 것은 JavaScript에서 자주 하는 변환 중 하나입니다.
화살표 함수는 함수를 더 간결하게 표현할 수 있고, 함수의 스코프를 명확하게 해줍니다.
화살표 함수는 function 키워드 대신 =>를 사용하여 정의됩니다.

이렇게 하면 함수가 더 간결해지고 가독성이 높아집니다.
화살표 함수는 주로 단순한 작업을 수행하는 함수를 정의할 때 사용되며, 코드를 간결하게 만들어줍니다.
그러나 주의할 점은 화살표 함수와 일반 함수는 this의 동작이 다르다는 점입니다. 

{
    // 선언적 함수
    func = () => {
        return "09. 화살표 함수 : 리턴값 함수";
    };
    console.log(func());

    //익명함수
    const func1 = () => {
        return "09. 화살표 함수 : 리턴값 함수";
    };
    console.log(func1());
}

10. 화살표 함수 : 매개변수 + 리턴값 함수

매개변수와 리턴값을 갖는 함수를 화살표 함수로 변환하는 것은 JavaScript에서 자주 하는 변환 중 하나입니다.
화살표 함수는 함수를 더 간결하게 표현할 수 있고, 함수의 스코프를 명확하게 해줍니다.

이렇게 하면 함수가 더 간결해지고 가독성이 높아집니다. 화살표 함수는 주로 간단한 작업을 수행하는 함수를 정의할 때 사용되며, 코드를 간결하게 만들어줍니다. 

{
    //익명함수
    const func = (str) => {
        return str;
    };
    console.log(func("10. 화살표 함수 : 매개변수 + 리턴값 함수"));

    //매개변수 하나이면 바로 생략 가능
    const func2 = str => {
        return str;
    };
    console.log(func2("10. 화살표 함수 : 매개변수 + 리턴값 함수"));

    //리턴 생략 가능
    const func3 = str => str;
    console.log(func2("10. 화살표 함수 : 매개변수 + 리턴값 함수"));

    //선언적 함수(가독성이 좋지않아서 잘안씀)
    func4 = str => str;
    console.log(func2("10. 화살표 함수 : 매개변수 + 리턴값 함수"));
}

11. 함수 유형 : 함수와 매개변수를 이용한 형태

JavaScript로 작성된 함수와 매개변수를 이용하는 방법에 대해 설명해드리겠습니다.
JavaScript에서 함수는 function 키워드를 사용하여 정의됩니다.
함수는 이름, 매개변수, 함수 본문으로 구성됩니다. 매개변수는 함수에 전달되는 값이며,
이 값은 함수 내에서 활용될 수 있습니다. 함수를 호출할 때는 매개변수에 전달할 값들을 인자로 전달합니다.

 {
    function func(num, str) {
        console.log(num + ". " + str);
        console.log(`${num}. ${str}`);
    }
    func(11, "함수가 실행되었습니다.");
}
{
    function func(x, y) {
        console.log(x, y);
    }
    func(11, "함수가 실행되었습니다.");
}

func 함수를 정의합니다. 이 함수는 두 개의 매개변수 num과 str을 받습니다.
함수 내부에서는 첫 번째 console.log 문장과 두 번째 ${num}. ${str} 문장을 사용하여 두 개의 문자열을 출력합니다.
첫 번째 방식은 문자열 연결 연산자(+)를 사용하여 구성하고, 두 번째 방식은 템플릿 리터럴을 사용합니다.

두 방식은 출력 결과가 동일합니다.
함수를 호출할 때, 매개변수로 숫자 11과 문자열 "함수가 실행되었습니다."를 전달합니다.
함수가 호출되면 매개변수 num에는 11이 할당되고, str에는 "함수가 실행되었습니다."가 할당됩니다.
함수 내부에서는 주어진 매개변수를 이용하여 두 개의 문장을 출력합니다.

12. 함수 유형 : 함수와 변수를 이용한 형태

JavaScript에서 함수와 변수를 이용하는 형태는 프로그램을 구성하는 중요한 요소입니다.
함수와 변수를 이용하여 데이터를 저장하고 처리하는 등 다양한 작업을 수행할 수 있습니다.
함수는 코드 블록을 정의하고, 이를 필요할 때 호출하여 실행할 수 있는 독립적인 단위입니다.

20. 함수 : 즉시실행 함수

즉시실행 함수(Immediately Invoked Function Expression, IIFE)는 자바스크립트에서 사용되는 함수의 패턴 중 하나입니다.
이 패턴은 함수를 정의함과 동시에 즉시 실행시키는 것을 의미합니다.
이것은 일반적으로 함수를 전역 스코프로 노출시키지 않고 지역 스코프 내에서 변수를 보호하고자 할 때 사용됩니다. 

{
    (function () {
        console.log("25. 함수가 실행되었습니다.");
    })();

    //화살표 함수 반영
    (() => {
        console.log("26. 함수가 실행되었습니다.");
    })();
}

이 구문에서 함수는 익명으로 선언되고, 그 다음에 즉시 실행됩니다.
함수를 괄호로 둘러싸는 것은 함수를 표현식으로 취급하고, 그런 다음 이 표현식을 실행함으로써 함수를 호출하는 것입니다.

즉시실행 함수는 다음과 같은 몇 가지 용도로 사용될 수 있습니다:
* 지역 스코프 생성: 즉시실행 함수 내에 선언된 변수는 해당 함수의 지역 스코프에서만 유효합니다. 이것은 전역 스코프를 오염시키지 않고 코드를 구성하는 데 도움이 됩니다.
* 모듈 패턴 구현: 즉시실행 함수는 모듈 패턴의 핵심 요소입니다. 모듈 패턴은 관련된 기능을 캡슐화하고 외부로부터 보호하기 위해 사용됩니다.
* 정보 은닉: 즉시실행 함수 내에서 선언된 변수는 외부에서 접근할 수 없으므로, 정보 은닉을 구현하는 데 사용될 수 있습니다.
* 초기화 코드 실행: 즉시실행 함수는 페이지 로드 시 초기화 코드를 실행하는 데 사용될 수 있습니다.

21. 함수 : 파라미터 함수

파라미터 함수(parameter function)는 다른 함수에 인자로 전달되는 함수를 의미합니다.
자바스크립트에서 함수는 일급 객체로 취급되기 때문에 함수를 다른 함수에 전달할 수 있습니다.
이때 전달되는 함수를 파라미터 함수라고 합니다. 

{
    function func(str = "27. 함수가 실행되었습니다.") {
        console.log(str)
    }
    func();

    const func1 = (str = "28. 함수가 실행되었습니다.") => {
        console.log(str)
    }
    func1();
}

파라미터 함수를 사용하면 다음과 같은 유연성과 장점을 얻을 수 있습니다: * 콜백 함수: 가장 흔한 사용 사례 중 하나는 비동기 작업에서 콜백 함수를 사용하는 것입니다.
예를 들어, 타이머 함수나 이벤트 핸들러에 콜백 함수를 전달하여 특정 조건이 충족될 때 실행되도록 할 수 있습니다.
* 콜백 기반 API와의 통합: 많은 라이브러리와 API는 비동기 작업을 처리하기 위해 콜백 함수를 요구합니다.
파라미터 함수를 사용하여 이러한 API와의 통합을 용이하게 할 수 있습니다.
* 유연한 동작 제어: 함수를 파라미터로 전달하면 호출되는 함수의 동작을 외부에서 제어할 수 있습니다.
예를 들어, 정렬 함수에 비교 함수를 전달하여 정렬의 방식을 사용자가 선택하도록 할 수 있습니다.
* 모듈화: 함수를 파라미터로 전달하여 다른 함수 내에서 사용되도록 함으로써 코드를 모듈화하고 재사용 가능한 구성 요소로 만들 수 있습니다.

22. 함수 : 재귀 함수

재귀 함수(recursive function)는 함수가 자기 자신을 호출하는 것을 허용하는 함수입니다.
이것은 반복적으로 자신의 몸체를 실행하여 원하는 결과를 얻는 데 사용됩니다.
재귀 함수는 명시적인 반복문을 사용하는 것보다 간결하고 직관적인 코드를 작성할 수 있게 해줍니다. 

{
    //01. 반복문을 사용하는 경우
    function func(num) {
        for (let i = 0; i < num; i++) {
            console.log("29. 함수가 실행되었습니다.");
        }
    }
    func(10);

    //02. 재귀 함수 사용하기
    function func1(num) {
        if (num < 1) return;

        console.log("30. 함수가 실행되었습니다.");
        func1(num - 1);    //재귀 호출
    }
    func1(10);
}

재귀 함수의 주요 특징은 다음과 같습니다:

* 자기 자신 호출: 재귀 함수는 함수 내에서 자기 자신을 호출합니다. 이것은 함수가 반복적으로 호출되면서 작업을 수행하는 것을 의미합니다.
* 종료 조건: 재귀 함수는 반드시 종료 조건을 가져야 합니다. 이 종료 조건이 없으면 함수가 무한히 호출될 수 있으므로 스택 오버플로우와 같은 문제가 발생할 수 있습니다.
* 스택 사용: 재귀 함수 호출은 호출 스택(call stack)을 사용합니다.
각 재귀 호출은 스택에 새로운 프레임을 추가하고, 종료 조건이 충족될 때까지 스택에 계속 쌓입니다. 그런 다음 종료 조건이 충족되면 스택에서 프레임이 제거됩니다.
* 재귀적 문제 해결: 재귀 함수는 일부 문제를 재귀적으로 해결하는 데 유용합니다.
예를 들어, 팩토리얼 계산, 피보나치 수열, 이진 트리 순회 등의 문제를 해결하는 데 사용될 수 있습니다.

재귀 함수는 코드를 간결하고 가독성 있게 만들 수 있는 강력한 도구이지만, 잘못 사용하면 스택 오버플로우와 같은 문제가 발생할 수 있으므로 주의해야 합니다.
또한, 반복문을 사용하는 것보다 재귀 함수가 더 많은 메모리와 실행 시간을 필요로 할 수 있습니다.
따라서 재귀 함수를 사용할 때는 재귀 호출의 깊이를 고려하여야 하며, 적절한 종료 조건을 반드시 설정해야 합니다.

23. 함수 : 콜백 함수

콜백 함수(callback function)는 다른 함수의 인자로 전달되어 특정 시점이나 조건에 따라 호출되는 함수를 말합니다.
자바스크립트에서 함수는 일급 객체이므로 함수를 변수에 할당하고 다른 함수에 전달할 수 있습니다.
콜백 함수는 이러한 특성을 활용하여 비동기적 작업이나 이벤트 처리 등에 사용됩니다. 

{
    //01. 이벤트
    function func() {
        console.log("31. 함수가 실행되었습니다.");
    }

    btn.addEventListener("click", func);

    //02. 함수를 다른 함수의 인자로 전달
    function func1() {
        console.log("32. 함수가 실행되었습니다.")
    }

    function func2(callback) {
        callback();
    }
    func2(func1);

    //03. 반복문을 사용하는 경우
    function func3(num) {
        console.log((num + ". 함수가 실행되었습니다."))
    }
    function func4(callback) {
        for (let i = 33; i < 38; i++) {
            callback(i)
        }
    }
    func4(func3);
}

콜백 함수의 특징은 다음과 같습니다:

비동기적 작업: 주로 비동기적 작업에서 사용됩니다. 예를 들어, 타이머 함수(setTimeout, setInterval)나 AJAX 요청, 파일 읽기 등의 작업에서 콜백 함수를 사용하여 작업이 완료되었을 때 실행되도록 할 수 있습니다.
이벤트 처리: 이벤트 핸들러에 콜백 함수를 등록하여 특정 이벤트가 발생했을 때 실행되도록 할 수 있습니다.
예를 들어, 버튼 클릭, 키보드 입력 등의 이벤트 처리에 콜백 함수를 사용할 수 있습니다.
비동기 코드의 순서 제어: 비동기적 작업에서 콜백 함수를 사용하여 작업이 완료된 후에 특정 코드가 실행되도록 할 수 있습니다.
이를 통해 비동기 코드의 실행 순서를 제어할 수 있습니다.
모듈화와 재사용성: 콜백 함수를 사용하여 코드를 모듈화하고 재사용 가능한 구성 요소로 만들 수 있습니다.
함수를 인자로 전달하여 다양한 동작을 수행하는 함수를 만들 수 있습니다.

콜백 함수는 자바스크립트에서 주요한 개념 중 하나이며, 비동기적 작업과 이벤트 처리 등 다양한 상황에서 사용됩니다.
콜백 함수를 올바르게 활용하면 코드를 더 유연하고 효율적으로 작성할 수 있습니다.
그러나 콜백 지옥(callback hell)과 같은 문제를 피하기 위해 콜백 함수를 사용할 때는 콜백 체이닝, Promise, async/await 등의
기술을 사용하여 코드를 더 깔끔하게 구성하는 것이 중요합니다.

26. 함수 : 비동기 방식 : asyne/await

async/await은 자바스크립트에서 비동기 코드를 보다 간결하게 작성할 수 있도록 해주는 문법입니다.
이를 사용하면 프로미스를 기반으로 하는 비동기 코드를 동기적으로 작성하는 것처럼 보이게 만들 수 있습니다.
이러한 문법은 ES8(ES2017)에서 도입되었습니다. 

{
    //01 :일반적인 함수(동기방식)
    function func() {
        console.log("45. 함수가 실행되었습니다.")
    }
    func();

    //02 : 함수(비동기 방식)
    async function func2() {
        console.log("46. 함수가 실행되었습니다.")
    }
    func2();

    //03 : 비동기방식(fetch())
    async function func3() {
        const result = await fetch("https://123dd654.github.io/class2024/json/gineungsaJC2005_02.json");
        const data = await result.json();
        console.log(data);
    }
    func3();

    //04 : 비동기 + 에러
    async function func4() {
        try {
            const result = await fetch("https://123dd654.github.io/class2024/json/gineungsaJC2005_02.json");
            const data = await result.json();
            console.log(data);
        } catch (error) {
            console.error(error)
        }
    }
    func4();
}

async/await은 다음과 같은 특징을 가지고 있습니다:

* 비동기 함수 선언: async 키워드를 사용하여 비동기 함수를 선언할 수 있습니다.
이렇게 선언된 함수는 항상 프로미스를 반환합니다.
* await 키워드: await 키워드는 프로미스가 이행될 때까지 기다리는 역할을 합니다.
